# atom01_deploy 部署框架技术文档

## 1. 项目含义

`atom01_deploy` 是 Roboparty 面向 RPO/Roboto 人形机器人的实物部署框架，基于 ROS2 Humble 构建。其核心职责是将仿真环境中通过强化学习训练得到的策略模型（ONNX 格式）部署到真实机器人主控上，完成从"仿真策略"到"实物运动"的闭环落地。

该框架直接对接真实硬件（达妙电机、HIPNUC IMU、手柄），在 Orange Pi 5 Plus / RDK X5 主控上运行，使用实时内核与 SCHED_FIFO 调度保证控制周期确定性。它不属于训练侧代码，而是训练产物（策略权重、动作序列）在真机上的执行载体。

## 3. 算法原理

### 3.1 整体控制架构

系统采用"感知—推理—控制"三段式架构，运行在三个实时线程上：

```
┌─────────────┐   ┌──────────────────┐   ┌──────────────────┐
│  感知层      │   │  推理层 (50Hz)    │   │  控制层 (250Hz)   │
│  IMU + 电机  │──▶│  ONNX 策略前向    │──▶│  PD + 闭链解耦    │
│  观测构造    │   │  动作平滑滤波     │   │  MIT 指令下发     │
└─────────────┘   └──────────────────┘   └──────────────────┘
       ↑                                          │
       └──────────── 反馈闭环 ────────────────────┘
```

- **主线程**（优先级 50）：ROS2 执行器，处理服务、订阅回调。
- **推理线程**（优先级 70，SCHED_FIFO）：周期 `dt * decimation = 0.004 * 5 = 20ms`（50Hz），执行 ONNX 前向。
- **控制线程**（优先级 70，SCHED_FIFO）：周期 `dt = 4ms`（250Hz），执行动作平滑与电机下发。

### 3.2 观测构造

策略输入观测由多个分段拼接而成，默认布局（[inference.yaml](src/inference/config/inference.yaml)）：

```
obs = [ang_vel(3), gravity_b(3), cmd_vel(3), dof_pos(23), dof_vel(23), last_action(23)]
```

各分段定义：

**角速度 `ang_vel`**：IMU 角速度经外参旋转到机体系，再乘以缩放系数。
$$\omega_{body} = R_{imu}^{body} \cdot \omega_{imu}$$

**重力向量 `gravity_b`**：世界系重力方向在机体系下的投影，用于表征姿态。
$$g_{body} = R_{world}^{body} \cdot \begin{bmatrix}0\\0\\-1\end{bmatrix} = q_{w2b} \cdot g_{world}$$

其中 $q_{w2b} = q_{b2w}^{-1}$，由 IMU 四元数求逆得到。当 $g_{body}.z > gravity\_z\_upper$（如 -0.5）时判定机器人摔倒，立即关机保护。

**指令速度 `cmd_vel`**：手柄或 `/cmd_vel` 话题输入的 $[v_x, v_y, \omega_z]$，经 `clip_cmd` 裁剪。

**关节位置 `dof_pos`**：当前关节角相对默认角的偏移，乘以缩放系数。
$$q_{obs,i} = (q_i - q_{default,i}) \cdot s_{dof\_pos}$$

**关节速度 `dof_vel`**：当前关节速度，乘以缩放系数。

**上一时刻动作 `last_action`**：策略上一帧输出（未经平滑的原始值），提供短期时序信息。

**帧堆叠 `frame_stack`**：将连续 10 帧观测沿时间维拼接，为策略提供历史时序信息，输入维度 $= 78 \times 10 = 780$。堆叠方式支持 `frame_major`（按帧排列）与 `obs_major`（按观测量排列）。

所有观测值最终经 `clip_observations`（默认 100.0）裁剪，防止数值爆炸。

### 3.3 策略前向推理

策略网络为标准 MLP，通过 ONNX Runtime 部署。前向过程：

$$a_{raw} = \pi_\theta(o_{stacked})$$

输出 23 维原始动作，经以下后处理转为目标关节角：

**动作裁剪 + 缩放 + 偏置**：
$$a_i = \text{clip}(a_{raw,i}, -c_{act}, c_{act}) \cdot s_{act} + q_{default,usd2urdf[i]}$$

其中 `usd2urdf` 是 USD 仿真关节序号到 URDF 实物关节序号的映射数组，保证仿真训练序号与实物电机序号对齐。

**动作低通滤波**（控制线程中执行，[inference_node.cpp#L219](src/inference/src/inference_node.cpp)）：
$$a_{smooth,t} = \alpha \cdot a_t + (1-\alpha) \cdot a_{smooth,t-1}$$

`act_alpha=1.0` 表示不滤波，直接下发；小于 1 时起平滑作用，降低高频抖动。

### 3.4 关节 PD 控制

非闭链关节采用 MIT 模式位置 PD 控制（[robot_interface.cpp#L162](src/inference/src/robot_interface.cpp)）：

$$\tau_i = k_{p,i}(q_{des,i} - q_i) + k_{d,i}(0 - \dot{q}_i)$$

MIT 指令格式：`motor_mit_cmd(pos, vel=0, kp, kd, torque=0)`，由电机端固件计算力矩。

`kp/kd` 按 23 个关节分别配置（[robot.yaml](src/inference/config/robot.yaml)），髋关节刚度最高（100~150），末端关节最低（20~40）。

### 3.5 踝关节闭链解耦

机器人踝关节采用四连杆闭链结构，两个电机（roll/pitch）通过连杆耦合驱动踝关节。直接对电机发位置指令无法解耦，需要运动学正反解（[close_chain_mapping.cpp](src/inference/src/utils/close_chain_mapping.cpp)）。

**机构参数**：

| 参数 | 含义 | 值 |
|------|------|----|
| `l_bar` | 摇臂长度 | 20 mm |
| `l_rod[0]` | 长连杆 | 180 mm |
| `l_rod[1]` | 短连杆 | 110 mm |
| `l_spacing` | 两侧间距 | ±42.35 mm |

**逆运动学**（关节角 → 电机角）：已知踝关节 roll $q_{roll}$ 与 pitch $q_{pitch}$，求解两个电机角度 $\theta_1, \theta_2$。

通过几何约束求解连杆闭环方程，得：
$$\theta_i = \arcsin\left(\frac{b \cdot c + \sqrt{b^2 c^2 - (a^2+b^2)(c^2-a^2)}}{a^2+b^2}\right)$$

其中：
$$a = r_{C,i}^x - r_{A,i}^x, \quad b = r_{A,i}^z - r_{C,i}^z, \quad c = \frac{l_{rod}^2 - l_{bar}^2 - \|r_C - r_A\|^2}{2 l_{bar}}$$

**正运动学**（电机角 → 关节角）：已知电机角 $\theta_{ref}$，采用 Newton-Raphson 迭代求关节角：

$$x_{k+1} = x_k + \alpha \cdot J_{motor2joint} \cdot (\theta_{ref} - \theta(x_k))$$

迭代至 $\|\theta_{ref} - \theta(x_k)\| < 10^{-3}$ 或达到 100 次上限，步长 $\alpha=0.5$。初值取上一次解，加速收敛。

**力矩/速度映射**：通过雅可比矩阵在电机空间与关节空间间转换：
- 速度：$\dot{q}_{joint} = J_{motor2joint} \cdot \dot{\theta}_{motor}$
- 力矩：$\tau_{motor} = J_{motor2joint}^T \cdot \tau_{joint}$

控制流程（[robot_interface.cpp#L113](src/inference/src/robot_interface.cpp)）：
1. 读取电机角速度力矩 → 正运动学转到关节空间
2. 在关节空间计算 PD 力矩
3. 逆运动学将力矩转回电机空间
4. 闭链电机以纯力矩模式下发（`motor_mit_cmd(0,0,0,0,tau)`）

### 3.6 多策略与模式切换

框架支持多策略并列配置，通过手柄 LB/RB 切换：

| 配置文件 | 策略 | 说明 |
|----------|------|------|
| [inference.yaml](src/inference/config/inference.yaml) | policy.onnx | 基础行走，10 帧堆叠 |
| [inference_amp.yaml](src/inference/config/inference_amp.yaml) | policy_amp.onnx | AMP 对抗运动先验，1 帧输入，步幅更大 |
| [inference_attn_enc.yaml](src/inference/config/inference_attn_enc.yaml) | policy_attn_enc.onnx | 带感知编码器，额外 187 维高程观测 |
| [inference_beyondmimic.yaml](src/inference/config/inference_beyondmimic.yaml) | policy.onnx + wave/dance/punch | 多策略：行走 + 动作模仿 |
| [inference_getup.yaml](src/inference/config/inference_getup.yaml) | policy.onnx + policy_getup.onnx | 行走 + 跌倒起身 |
| [inference_interrupt.yaml](src/inference/config/inference_interrupt.yaml) | policy_interrupt.onnx | 上半身可外部中断控制 |

**beyondmimic 模式**：配置多个策略，第 0 个为行走主策略，其余为动作模仿策略（配合 .npz 动作序列）。LB 键切换"行走/动作"模式，RB 键在多个动作序列间循环选择。动作策略观测包含 `motion_pos` 与 `motion_vel`，即当前动作序列的目标关节角与速度，策略学习如何平滑跟踪该动作。

**interrupt 模式**：观测追加 1 维 `interrupt` 标志位。当标志为 1 时，上半身 10 个关节（手腕、肘、肩）由外部 `/joint_ref_states` 话题直接接管，下半身仍由策略控制。用于实现"行走中手动控制手臂"等场景。

### 3.7 实时性保障

- **实时内核**：Orange Pi 5 Plus 安装 5.10 RT 内核（assets 中的 deb 包），RDK X5 预装。
- **内存锁定**：`mlockall(MCL_CURRENT | MCL_FUTURE)` 锁定进程内存，禁止换页。
- **实时调度**：推理/控制线程设置 `SCHED_FIFO`，优先级 70；主线程优先级 50；线程池优先级 70。
- **权限配置**：`/etc/security/limits.conf` 中设置 `rtprio 98` 与 `memlock unlimited`。
- **周期补偿**：每轮记录耗时，精确 `sleep_for` 剩余时间；超周期时打印告警。

## 4. 关键参数说明

### 4.1 推理参数（inference.yaml）

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `joint_num` | 23 | 关节数 |
| `dt` | 0.004 | 控制周期（s），250Hz |
| `decimation` | 5 | 推理降采样倍数，推理频率 = 250/5 = 50Hz |
| `frame_stacks` | 10 | 观测帧堆叠数 |
| `action_scale` | 0.25 | 动作缩放系数 |
| `act_alpha` | 1.0 | 动作低通滤波系数，1.0 表示不滤波 |
| `clip_actions` | 100.0 | 动作裁剪范围 |
| `clip_observations` | 100.0 | 观测裁剪范围 |
| `gravity_z_upper` | -0.5 | 摔倒判定阈值，$g_z$ 超过此值则关机 |
| `usd2urdf` | [...] | USD 关节序号 → URDF 关节序号映射 |

### 4.2 关节默认角（joint_default_angle）

23 维默认关节角（rad），策略输出在此基础上叠加。典型值：
- 髋关节 pitch：-0.1 / 0.3 / -0.2（屈膝准备姿态）
- 膝关节：0.78（微屈）
- 手腕：0.18（自然下垂）

### 4.3 关节限位（joint_limits）

23 组 $[min, max]$ 区间。`inference.yaml` 使用严格限位（基于机械结构），`inference_amp/getup` 等使用宽松限位（±3.14），因为起身/大动作需要更大运动范围。超限即触发关机保护。

### 4.4 PD 增益（robot.yaml）

`kp` 范围 20~150，`kd` 范围 1.0~5.0。髋/膝等承重关节刚度大，手腕/肘等末端关节刚度小。

## 5. 运行步骤

### 5.1 环境准备

1. 安装 ROS2 Humble 及依赖：
```bash
sudo apt update && sudo apt install -y ccache libfmt-dev libspdlog-dev libeigen3-dev screen python3-yaml python3-numpy
sudo apt install -y ros-humble-joy
```

2. 拉取代码（含子模块）：
```bash
git clone --recursive https://github.com/Roboparty/roboparty_deploy.git
cd roboparty_deploy
git submodule update --init --recursive
```

3. Orange Pi 5 Plus 安装实时内核，并配置 `limits.conf` 授予 `rtprio 98` 与 `memlock unlimited` 权限，重启后用 `ulimit -r` 验证输出 98。

4. 配置 udev 规则绑定 CAN/IMU 设备，避免插拔顺序影响：
```bash
sudo cp assets/99-auto-up-devs-orangepi.rules /etc/udev/rules.d/
sudo udevadm control --reload && sudo udevadm trigger
```

### 5.2 电机标零

首次使用或零点丢失时执行。两种方式：

**方式一：脚本逐个标零**（推荐首次装机使用）
```bash
colcon build --symlink-install
source /opt/ros/humble/setup.bash && source install/setup.bash
python3 scripts/set_zero.py
```
按提示手动摆位，Enter 标零，空格跳过。脚本将电机切阻尼模式便于转动。

**方式二：服务批量标零**（软件已启动时）
```bash
./tools/start_robot.sh
ros2 service call /init_motors std_srvs/srv/Trigger
ros2 service call /set_zeros std_srvs/srv/Trigger
```

腰部 yaw 标零注意：转至限位块标零保留 `robot.yaml` 中 `motor_zero_offset` 的 2.093；用打印件固定标零则改为 0.0。

### 5.3 编译与启动

```bash
./tools/start_robot.sh
```

该脚本自动执行 `colcon build --symlink-install`，并在后台启动两个 screen 会话：
- `inference_session`：推理节点
- `joy_session`：手柄节点

查看输出：`screen -r inference_session`
停止会话：`screen -S inference_session -X quit`

### 5.4 切换策略

修改 [inference.launch.py](src/inference/launch/inference.launch.py) 中加载的配置文件名，重新运行 `./tools/start_robot.sh`。

可用的推理配置文件：

|----------|------|------|
| `inference.yaml` | `policy.onnx` | 基础行走策略，10 帧堆叠，手柄速度指令控制 |
| `inference_amp.yaml` | `policy_amp.onnx` | AMP 对抗运动先验策略，1 帧输入，步幅更大、风格更自然 |

## 6. 安全机制

1. **摔倒保护**：每帧检查 $g_{body}.z > gravity\_z\_upper$，触发即关闭节点。
2. **关节超限保护**：`get_dof_pos_obs` 中检查每个关节是否超出 `joint_limits`，超限即关机。
3. **电机离线检测**：`apply_action` 中检查 `response_count > offline_threshold(25)`，判定电机离线抛异常。
4. **推理暂停联动**：标零、复位等服务在推理运行时拒绝执行；切换策略时自动暂停推理。
5. **动作裁剪**：观测与动作均经 `clip` 裁剪，防止数值异常传入策略或电机。

## 7. Python SDK

编译后通过 `install/setup.bash` 暴露三个模块：

- `imu_py`：IMU 驱动，`create_imu` → `get_quat/get_ang_vel/get_lin_acc`
- `motors_py`：电机驱动，`create_motor` → `init_motor/motor_mit_cmd/get_motor_pos`
- `robot_py`：机器人接口，`RobotInterface(config)` → `init_motors/apply_action/get_joint_q`

示例见 [scripts/imu_py_example.py](scripts/imu_py_example.py)、[scripts/motors_py_example.py](scripts/motors_py_example.py)。动作序列回放见 [scripts/motion_player.py](scripts/motion_player.py)。
